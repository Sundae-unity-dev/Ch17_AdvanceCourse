# 심화 모듈: 상호작용·소통·캐릭터 표현 동기화

> 학생이 기본 과정에서 만든 Quantum 3 멀티플레이 캐릭터 위에 **음성·채팅·이모트·자연스러운 움직임 표현** 을 붙여 "사람이 만나면 자연스럽게 존재감을 느끼고 소통하는 공간" 으로 확장하는 모듈.

---

## 1. 모듈 개요

- **분량**: 2주 (수업 1회 + 실습 11회)
- **전제 조건 (Prerequisites)**
  - 기본 과정 (01~05장) 수료
  - Unity 2022.3 LTS 이상 사용 경험
  - Quantum 3 멀티플레이어 캐릭터 게임을 한 번 빌드해 본 상태
  - C# 기본 문법, 이벤트·코루틴·델리게이트 이해
- **베이스 프로젝트**: 기본 과정 챕터 05 결과물 (Quantum 3 + PlayerCharacter + 멀티 빌드 동작 상태)

## 2. 학습 목표 (Learning Outcomes)

이 모듈을 끝낸 학생은 다음을 할 수 있다:

1. **Photon Voice 2** SDK 를 프로젝트에 통합하고 AppId 발급·연결 흐름을 설명할 수 있다
2. **Recorder · Speaker · VoiceConnection** 구성요소의 역할을 구분하고 적절한 위치에 배치할 수 있다
3. **Push-to-Talk** 방식의 음성 송신 시스템을 구현할 수 있다
4. 거리 기반 **Spatial Audio (3D 음향)** 를 적용해 가까이 가면 잘 들리는 효과를 구현할 수 있다
5. **RPC 기반 텍스트 채팅** 시스템을 설계·구현할 수 있다 (Photon Realtime 의 RaiseEvent 또는 RPC 활용)
6. 키 입력으로 트리거되는 **이모트 애니메이션 동기화** 를 구현할 수 있다
7. **풀바디 포즈 (앉기·기대기·눕기)** 를 Animator Layer + Avatar Mask 로 구현하고 동기화할 수 있다
8. **부드러운 이동 보간 (Lerp/Slerp)** 과 **외삽 (Extrapolation)** 을 적용해 네트워크 지연을 가리는 자연스러운 캐릭터 이동을 구현할 수 있다
9. **Blend Tree** 를 사용해 속도별 정지→걷기→달리기 자연 전환 + 동기화를 구현할 수 있다
10. **Footstep IK** 와 **LookAt 시선 처리** 로 캐릭터가 지면·다른 플레이어와 자연스럽게 상호작용하도록 구현할 수 있다
11. 두 플레이어가 동시에 참여하는 **소셜 제스처 (악수·하이파이브)** 를 동기화 구현할 수 있다
12. 멀티플레이어 환경에서 발생하는 음성·채팅·표현·움직임 동기화의 **네트워크 비용·지연·동기화 한계** 를 비교하고 트레이드오프를 설명할 수 있다
13. **각 실습의 응용 질문을 자기 식으로 풀어** 자기 프로젝트로 확장할 수 있다

## 4. 이론 수업자료 상세

### 섹션 1: 멀티플레이 통신 패턴 복습 (4~6p)
- State Sync · Input Sync · Event 기반 RPC 비교
- 결정론적 시뮬레이션이 음성·채팅을 처리하지 않는 이유 (비결정론 데이터)
- Photon 솔루션 분리: Quantum 3 (시뮬레이션) + Photon Realtime 5 (음성·이벤트)

### 섹션 2: Photon Voice 2 아키텍처 (8~10p)
- VoiceClient · VoiceConnection · Recorder · Speaker 역할
- 음성 데이터 흐름: 마이크 → Encoder (Opus) → Photon Cloud → Decoder → AudioSource
- 자체 서버 (Photon Server) vs Photon Cloud 비교
- 비용 모델 (CCU, 트래픽)

### 섹션 3: Push-to-Talk vs Open Mic (4~6p)
- 사용 시나리오 비교 (게임 보이스 vs 메타버스 상시 통화)
- 입력 모달 (키보드 키, 키 토글, 음성 활성화 VAD)
- 마이크 권한 처리 (Windows/Mac/Mobile)

### 섹션 4: Spatial Audio (3D 음향) (4~6p)
- AudioSource 의 3D 설정 (Min/Max Distance, RolloffMode)
- HRTF / Steam Audio 와의 차이 (Photon 기본은 단순 거리 감쇠)
- 가청 거리 vs 컬링 거리

### 섹션 5: 채팅 동기화 패턴 (4~6p)
- Photon Chat (별도 채널) vs RPC 기반 (RaiseEvent)
- 채팅 채널 설계 (전체·팀·1:1·근접 채팅)
- 메시지 영속성 (저장 필요 시 백엔드 DB 연동)

### 섹션 6: 이모트·UI 동기화 패턴 (4~6p)
- 애니메이터 트리거 동기화 (PUN Animator View vs RPC 트리거)
- UI 동기화 vs 로컬 표시 (이모트 버튼은 로컬, 애니메이션은 모든 클라이언트)
- 네트워크 비용 절감 패턴 (이모트 ID 정수 1byte로 전송)

### 섹션 7: 캐릭터 표현 — 풀바디 포즈와 Blend (10~12p)
- **Animator Layer + Avatar Mask** — 상체·하체 분리해서 손은 흔들면서 다리는 걷기
- **풀바디 포즈 (앉기·기대기·눕기)** — Trigger vs Bool 동기화, 진입·종료 처리, 두 명이 같은 의자에 앉으려 할 때 충돌
- **Blend Tree** — 1D (속도) vs 2D (속도 + 방향), Threshold 설정, 동기화 시 Blend Parameter 전송 비용
- **State Machine 동기화** — Animator parameter 전송 vs State 이름 전송, 결정론적이지 않은 이유 (Crossfade 타이밍 차이)

### 섹션 8: 자연스러운 움직임 — 보간·IK·시선 (12~14p)
- **왜 네트워크 캐릭터는 끊겨 보이나** — 패킷 전송 주기 (10~30Hz) vs 화면 갱신 주기 (60Hz+) 의 간극
- **보간 (Interpolation)**: 두 수신 패킷 사이를 Lerp/Slerp 로 채우기. 지연 1~2 tick 발생하지만 부드러움
- **외삽 (Extrapolation / Dead Reckoning)**: 마지막 속도·방향으로 미래 위치 예측. 패킷 끊김에 강하지만 오차 위험
- **언제 무엇을 쓸까**: 같은 방 안 ↔ 가까운 거리 (보간), 빠른 액션 게임 (외삽 + 후속 보정)
- **Footstep IK** — Animation Rigging 패키지 활용, RaycastFromFoot 로 지면 추적, 경사·계단에서 발 미끄러짐 방지
- **LookAt 시선 처리**: 머리만 회전 vs 풀바디 회전, Animation Rigging MultiAimConstraint, 거리·각도 임계값
- **소셜 제스처 동기화 패턴**: 한쪽이 트리거 → RPC 로 상대방에 신호 → 양쪽이 같은 시점에 애니메이션 시작 + 두 캐릭터 위치 정렬 (예: 악수 위치로 살짝 이동)
- **네트워크 비용·지연 트레이드오프 요약 표**

## 8. 응용 질문 패턴 — "여기서 이렇게 하면 어떻게 구현해요?"

각 실습 끝에 1페이지 분량의 **"🚀 응용 질문 (도전 과제)"** 섹션을 두어 학생의 상상력을 자극한다. 강사는 답을 직접 주지 않고 "어떤 접근이 가능할까?" 식으로 학생이 스스로 설계하도록 유도.

**좋은 응용 질문의 특징**

- 정답이 한 가지로 정해져 있지 않음 (탐색 여지)
- 본 실습 지식 위에 1~2단계 확장하면 풀림 (난이도 적정)
- 결과물이 시각적으로 즐거움 (학생이 만들고 싶어짐)
- 다른 모듈·다른 분야와 연결됨 (예: UI / 사운드 / AI)

**예시**

> [L3 Spatial Audio]
> "벽 뒤에 있는 사람의 목소리가 작게 들리거나 막히게 하려면 (오클루전), 어떤 컴포넌트와 어떤 계산이 필요할까요?"
>
> 힌트: Physics.Raycast 로 캐릭터 사이에 장애물 검출 → AudioSource 의 spatialBlend·dopplerLevel 조정. Steam Audio 같은 외부 솔루션도 검토.

> [L7 이동 보간 + Blend Tree]
> "캐릭터마다 다른 걷기 스타일(예: 무거운 캐릭터는 흔들리며 걷기)을 동기화하려면, 같은 입력에 대해 다른 결과를 어떻게 보여줄까요?"
>
> 힌트: 캐릭터 데이터에 walkStyle 파라미터, Animator 의 Override Controller, Blend Tree 의 Animation Clip 교체.

응용 질문은 **평가 항목**에도 들어가 있어 (10%), 학생이 1개 이상 구현해서 제출해야 만점.

## 9. 강사용 메모

- **공통 함정**: 학생들이 Voice SDK 임포트 시 AppId 를 빈 채로 두는 경우가 많음 → 실습 1 마지막에 AppId 입력란 강조
- **마이크 권한**: Windows 보안 설정에서 Unity 가 마이크 접근 권한 없는 경우 있음 → 권한 확인 단계 안내 필요
- **테스트 전제**: 강의실 PC에서 두 명이 접속해야 검증 가능 — 한 명이면 빌드 + Editor 동시 실행으로 테스트
- **확장 아이디어 (다음 모듈 연결)**: 채팅에 욕설 필터링·음성 녹화 저장 등은 "운영" 모듈로 미룸
